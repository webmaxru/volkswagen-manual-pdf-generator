## Pre-built manuals

If you just want to read the manual, ready-made PDFs live in [`manual/`](manual/):

- 🇳🇴 Norwegian (`nb`) — [`manual/nb/`](manual/nb/)
- 🇬🇧 English (`en`) — [`manual/en/`](manual/en/)

The rest of this document is only relevant if you want to (re)generate them yourself.

---

Produces a single, searchable PDF (with embedded illustrations) of the Volkswagen Digital Manual for the ID.4. Defaults match a Norwegian ID.4 / ID.4 GTX (part number `11A012777AR`, brand pattern `WVW`), but every parameter can be overridden on the command line.

The manual is only published online on volkswagen.no. This tool fetches it and renders it offline as one PDF you can search, annotate and print.

> **Norwegian-market only.** This project intentionally targets cars sold in the Norwegian market. The script always fetches from `volkswagen.no` regardless of language — it never hits other national VW portals and never machine-translates content. The same Norwegian backend (region `RDW`) serves all the languages we support; we just ask it for a different `--lang` / `--ownersManualTerm`. There is a hard guard in `fetch_manual.js` that refuses to run if `--landing` points anywhere other than `volkswagen.no`.

## Quick start

```powershell
# one-time
npm install

# Norwegian (default)
npm run fetch:nb

# English (sourced from volkswagen.no, NOT from a UK/US site)
npm run fetch:en

# Russian (sourced from volkswagen.no)
npm run fetch:ru
```

Output goes into `manual/<lang>/` (e.g. `manual/nb/`, `manual/en/`):

- `manual/<lang>/<brand>_<model>_<partnumber>_<lang>.pdf` — the final searchable PDF (A4, cover + TOC + page numbers).

Naming is unified across languages: `<brand>_<model>_<partnumber>_<lang>.pdf`, all lowercase. With current defaults this is `volkswagen_id.4_11a012777ar_<lang>.pdf` (e.g. `volkswagen_id.4_11a012777ar_nb.pdf`). The `--brand` and `--model` flags only affect this filename — they don't change anything fetched from the backend.

That's it — only the PDF is kept. The intermediate HTML and JSON dumps used during a run are written to a temp folder / cleaned up automatically.

## Image optimization

Every raster image embedded in the final PDF is stored as **JPEG** — no PNG/Flate-encoded raster streams. This happens in two passes:

1. **Inline pass** — `fetch_manual.js` re-encodes each fetched raster with `sharp` (mozjpeg, default `q=80`, max width 1600 px, white background flatten) before inlining as a `data:` URI. SVGs are kept as vector.
2. **PDF post-process** — `optimize_pdf.py` runs automatically after `page.pdf()` and rewrites every PNG image XObject in the produced PDF as JPEG using `pikepdf` + `Pillow`. This step is **mandatory**: Chromium's print pipeline silently re-rasterizes embedded `<img>` elements as PNG, undoing pass 1; without pass 2 the PDF is ~5× larger.

Combined effect: a regenerated language manual goes from **~73 MB → ~13 MB** (≈83 % saved) with no visible quality loss. Tune with `--jpegQuality` and `--jpegMaxWidth` (forwarded to both passes). See AGENTS.md → **Image optimization** for the full rationale.

## Updating the manual

Volkswagen occasionally publishes a new edition (the "Utgave 09.2025" line on the cover page is the version marker). Just re-run the same command — the backend always returns the latest topic IDs for your part number.

## Any other language

DIMA exposes the manual in many languages, all served from the same Norwegian backend. Use the generic form, but **always keep `--landing` on `volkswagen.no`**:

```powershell
node fetch_manual.js `
  --lang <short> `
  --market NO `
  --locale nb-NO `
  --region RDW `
  --ownersManualTerm "<localized-manual-title>" `
  --landing https://www.volkswagen.no/no/min-volkswagen/digital-instruksjonsbok-innhold.html
```

The output filename is auto-derived as `manual/<lang>/<brand>_<model>_<partnumber>_<lang>.pdf` (lowercase). Override only via `--out` if you really need a custom path.

See AGENTS.md for the table of valid `ownersManualTerm` values per language.

## CLI flags

| Flag | Default | Notes |
|---|---|---|
| `--partNumber` | `11A012777AR` | Printed inside the original PDF; also visible on the manual's URL on volkswagen.no. |
| `--brandPattern` | `WVW` | First three chars of the VIN. |
| `--lang` | `nb` | BCP-47 short code, e.g. `nb`, `en`, `de`, `sv`, `fi`, `fr`, `es`, `it`, `nl`, `pl`, `cs`, `pt`, `tr`, `ja`, `ko`… |
| `--market` | `NO` | ISO 3166 country code. Keep at `NO`. |
| `--region` | `RDW` | Internal VW region code. Norway is `RDW`. |
| `--locale` | `nb-NO` | Browser locale (only affects the cookie banner language). |
| `--landing` | VW Norway "innhold" page | **Must be on `volkswagen.no`** — the script will refuse to run otherwise. |
| `--ownersManualTerm` | `Instruksjonsbok` | The localized title of the owner's manual. Examples: `Owner's Manual` (en), `Betriebsanleitung` (de), `Manuel d'utilisation` (fr). |
| `--tenant` | `default` | Almost always `default`. |
| `--concurrency` | `8` | Parallel topic fetches. |
| `--jpegQuality` | `80` | mozjpeg quality (1–100) for raster images. |
| `--jpegMaxWidth` | `1600` | Max width in px before JPEG re-encode (no upscaling). |
| `--brand` | `volkswagen` | Used **only** in the output filename. |
| `--model` | `id.4` | Used **only** in the output filename. |
| `--out` | `manual/<lang>/<brand>_<model>_<partnumber>_<lang>` (lowercase) | Output basename. Parent directories are created automatically. |

## Requirements

- Node 18 or newer (tested on Node 22).
- `npm install` downloads Playwright + bundled headless Chromium (~140 MB) and `sharp`.
- Python 3.9+ with `pikepdf` and `Pillow`: `pip install pikepdf Pillow`. Used by `optimize_pdf.py` to re-encode PDF image streams as JPEG (mandatory — see "Image optimization"). The build will fail if Python or these modules are missing rather than silently producing an oversized PDF.
- Internet access to `volkswagen.no`, `prod-a.dima.fan.vwapps.run`, `userguide.volkswagen.de` and `cdn.dima.fan.vwapps.run`.

## Troubleshooting

- **"Cannot resolve topicId for this language"** — your `--ownersManualTerm` is probably wrong for the language. Look it up in AGENTS.md.
- **"--landing must be on volkswagen.no"** — the Norwegian-market guard kicked in. Don't override `--landing` to another country's portal.
- **PDF is missing images** — re-run; the asset CDN occasionally 503s a single file.
- **All topics fail to fetch with 403** — the token format or backend contract has probably changed. See `AGENTS.md` for the reverse-engineering procedure.
